# Contributing to Talocode

Talocode is open-source-first. Contributions are welcome across all repositories.

## Ways to Contribute

- **Report bugs** — open an issue in the relevant product repo with reproduction steps
- **Improve docs** — fix typos, clarify explanations, add examples
- **Write examples** — show how Talocode products work together in real workflows
- **Build integrations** — connect Talocode APIs to your favorite tools and frameworks
- **Submit PRs** — bug fixes, features, and improvements are welcome

## Getting Started

1. Fork the repository you want to contribute to
2. Create a feature branch (`git checkout -b feat/your-change`)
3. Make your changes
4. Run existing tests
5. Submit a pull request

## Guidelines

- Keep commit messages clear and descriptive
- Match the existing code style — no blanket reformatting
- Add tests for new functionality
- Update docs when behavior changes
- Do not commit secrets, API keys, or credentials

## Code of Conduct

Be respectful, constructive, and inclusive. Talocode is built by a global community of developers.

## Questions

Open a discussion or issue in the relevant repository.
